# RTG 大车防撞 BEV 感知系统——总体技术方案

> 版本: v0.2 (修订)
> 日期: 2026-05-21 (修订), 初稿 2026-05-15
> 基于: [大车防撞项目知识文档](./大车防撞项目知识文档.md) 中已确认的技术决策

---

## 1. 项目概述

### 1.1 项目目标
为轮胎式集装箱门式起重机（RTG，轮胎吊）构建面向大车防撞的 BEV 感知系统。系统输出：
1. 标准化 3D/BEV 障碍物检测结果。
2. 分级碰撞风险/防撞预警信息。

### 1.2 项目边界
- 纯感知项目，不接入 RTG 控制系统。
- 不获取 RTG 运动控制状态（轮速、编码器、IMU、GPS 等），需从传感器数据推断 RTG 运动状态。
- 大车静止时不主动预警，仅检测。
- 最终交付为产品级系统（稳定在线、标准接口、可视化、日志、异常恢复）。

### 1.3 验收方式
- **功能演示/现场测试用例通过率优先**，同时兼顾离线测试指标。
- mAP/NDS 使用项目自定义类别和距离范围，不绑定 NuScenes 官方评测。
- 盲区仅统计防撞有效 ROI 内，不按 360° 全域面积。

## 2. 场景与传感器

### 2.1 堆场几何（已修正）
- 标准化矩形堆场。
- **大车道与集卡车道平行**，均沿 RTG 行驶方向。集卡车道紧邻大车道。
- RTG 跨距约 23.5 m，轮组间距 ~12.0 m。
- 跨距范围内包含：六条集装箱道 + 一条集卡车道。
- RTG 每侧前后支腿通过连接梁连成一体。
- 地面有稳定车道黄线，可作为几何先验。

### 2.2 传感器配置

| 位置 | 相机 | 激光雷达 |
|---|---|---|
| 集卡侧前支腿 | 海康 DS-2XC64ZZY-SHZJ-4mm, 高~4m | OS1 128线, 10Hz, 高~1.5m |
| 集卡侧后支腿 | 同上 | OS1 128线, 10Hz, 高~1.5m |
| 禁行侧前支腿 | 同上 | RoboSense 32线, 旋转90°安装, 高~1.5m |
| 禁行侧后支腿 | 同上 | RoboSense 32线, 旋转90°安装, 高~1.5m |

- 相机: 竖屏1080×1920, 俯视图FOV锥角 46°, 87.6°为沿x纵深+高度方向, 斜向下朝向大车道。
- OS1: 360° 水平, 42.4° 垂直, 前方 180° 可用。
- RoboSense 32线: 旋转 90° 安装 → 垂直 360°, 水平 ~70°, 沿大车道方向扫描。
- 四相机之间基本无共视，四雷达硬同步。

### 2.3 防撞对象与类别体系

| 类别 | 说明 |
|---|---|
| `person` | 人员 |
| `truck` | 集卡（整体标注，不拆牵引车/挂车/载箱） |
| `car` | 乘用车/小型车辆 |
| `other_obstacle` | 邻车 RTG、AGV/IGV、叉车、锥桶、路障等 |

## 3. BEV 空间定义

### 3.1 坐标系
- **原点**: 集卡侧 OS1 128 线雷达在地面的垂直投影。
- **x 轴**: 沿大车道方向（C1 朝向），RTG 可沿 ±x 双向行驶。
- **y 轴**: 沿跨距方向（垂直于大车道），**从禁行侧指向集卡侧为正 (+y)**。
- **z 轴**: 竖直向上。
- 固定车体系，不随行驶方向翻转。
- 方向约定：使用 **+x/-x** 代替"前/后"（因为前/后随行驶方向变化），使用 **集卡侧（y≈0）/ 禁行侧（y≈-23.5m）** 代替"左/右"。

### 3.2 BEV 空间范围

| 轴 | 范围 | 说明 |
|---|---|---|
| x | [-60, 60] m | 沿大车道，前后各 60 m |
| y | [-25, 25] m | 跨距方向，含集卡车道 + 集装箱堆场 + 禁行侧 |
| z | [-3, 8] m | 近地至载箱高度 |

- 训练范围略大（60m），预警 ROI 精确裁剪。

### 3.3 RTG 自车 Footprint

RTG 不同于常规车辆，地面上仅有 4 条支腿。**拆分为两个独立矩形 footprint**：

```
        +y ↑ (跨距方向，从禁行侧指向集卡侧，与 ROS +y=left 对齐)

  y≈8.5╔══ 相邻箱区+禁行侧+集卡车道 ═════════════════════╗
       ┌── 中间车道 (两箱区间) ──────────────────────────┐
       ├── 6列集装箱+集卡车道 (相邻箱区) ───────────────┤
  y≈0 ╔══ 大车道(集卡侧)+C1+C2+L1原点+L2+集卡侧footprint ══╗
       │  原点 ● = L1 在大车道中间                       │
       │  集卡车道+6列集装箱+大车道(禁行侧)              │
       ▼  C3+C4+L3+L4+禁行侧footprint                    │
      -y ↓  (禁行侧方向)
        
  x轴 → 大车道方向 (大车道 ∥ 集卡车道)
```

- 两个 footprint 之间为正常检测空间（不画自车）。
- 支腿精确尺寸由现场测量提供。
- 自车 footprint 用于：从检测结果中排除自车结构、定义碰撞风险区域。

## 4. 模型架构

### 4.1 总体架构

```
传感器层:
  集卡车道侧: 2× OS1 128线雷达 + 2× 海康工业相机
  禁行侧:     2× RoboSense 32线雷达 + 2× 海康工业相机

感知层 (第一版):
  主模型: CenterPoint LiDAR-only BEV 检测
    输入: 集卡侧 2 雷达拼接点云(单帧)
    输出: person / truck / car / other_obstacle 3D 框
  辅助模型: 禁行侧延后 (第一版不实现)

后处理层:
  目标跟踪: 短时跟踪 (内部稳定 ID，对外可选)
  运动估计: 静态背景点云 ICP → 运动/静止 + 方向
  预警模块: 分级预警 (危险单帧触发 + 普通连续帧确认)

输出层:
  ROS1 topics: detections / tracks / warnings / ego_motion / diagnostics
```

### 4.2 CenterPoint 配置要点

| 配置项 | 值 (RTG 场景) |
|---|---|
| 模型框架 | det3d (CenterPoint 官方) |
| 检测器 | VoxelNet |
| Backbone | SpMiddleResNetFHD (SparseConv3D) |
| Neck | RPN (FPN) |
| 检测头 | CenterHead (Heatmap + Regression) |
| 激光雷达 | 2× 128线拼接 (外参拼接为统一单帧点云) |
| point_cloud_range | [-60,-25,-3, 60,25,8] |
| voxel_size | [0.075, 0.075, 0.2] |
| 类别数 | 4 (person, truck, car, other_obstacle) |
| sweeps | 单帧 (RTG 低速 + 128线密度足够) |

### 4.3 预训练权重策略
- 加载 NuScenes pretrained CenterPoint checkpoint (backbone + neck + head)。
- 整体 fine-tune RTG 数据集 (~4500 帧训练 + ~500 帧验证)。

### 4.4 训练配置
- Optimizer: Adam, one_cycle LR schedule, ~20 epochs。
- Batch size: 2 per GPU (RTX 3060 12GB)。
- 根据实际训练曲线决定是否调整。

## 5. 数据方案

### 5.1 数据流

**开发模式（离线训练）:**
```
ROS1 rosbag → rosbag_extract 工具 → 图像(.jpg) + 点云(.bin) + 时间戳
  → info.pkl 生成 → CenterPoint DataLoader → 训练
```

**部署模式（在线推理）:**
```
ROS1 topic 订阅 → 实时点云 → 体素化 → CenterPoint 推理
  → 后处理 → ROS1 topic 发布
```

### 5.2 标注方案

| 项目 | 方案 |
|---|---|
| 工具 | CVAT / LabelCloud 等开源工具 |
| 格式 | 点云 3D 框 (x,y,z,w,l,h,yaw) + 类别 |
| yaw | 标注工具旋转框设置，RTG 场景中集卡 yaw 基本为 0°/180° |
| 2D 标注 | 由 3D 框 + 内外参自动投影生成，不人工标注 |
| 遮挡/截断 | 不标注（RTG 场景遮挡简单） |
| 数量 | ~4500 训练 + ~500 验证 |
| 范围 | 仅集卡车道侧 2 相机 + 2 雷达覆盖区域 |
| 连续序列 | 5~10 个片段，每个 50~100 帧，覆盖集卡完整驶过过程 |

### 5.3 数据采集
- 正常作业场景为主 (>80%)，困难帧迭代采集补足。
- 需向现场或甲方提出：支腿坐标、大车道宽度、集卡车道宽度、传感器安装位置等测量需求。

## 6. 预警逻辑

### 6.1 ROI 配置策略

利用"相机固定 + 车道黄线在图像中位置固定"的强先验：

1. 在每个相机图像中一次性标注黄线/车道边界像素位置 (静态配置)。
2. 通过相机内外参投影到地面平面 (z=0) → BEV 车道边界。
3. 融合两个相机的 BEV 边界 → 统一大车道核心 ROI。
4. 预警模块用该 BEV ROI 过滤检测结果。

### 6.2 分级预警

| 等级 | 含义 | 触发方式 |
|---|---|---|
| 0 | 无风险 | — |
| 1 | 提示/关注 | 连续 2~3 帧确认 |
| 2 | 警告 | 连续 2~3 帧确认 |
| 3 | 危险 | 单帧即时触发 |

### 6.3 初版预警阈值

| 目标类别 | 危险 (单帧) | 警告 | 提示 |
|---|---|---|---|
| 人员 | 0~8 m | 8~15 m | 15~25 m |
| 集卡/车辆 | 0~12 m | 12~25 m | 25~40 m |
| 大型/未知障碍物 | 0~10 m | 10~20 m | 20~35 m |

附加规则：
- 仅对大车道核心 ROI 触发高等级预警。
- 横向入侵区可降低一级。
- RTG 静止时完全不预警，仅输出检测结果。
- 所有阈值和 ROI 配置化，支持现场调整。

### 6.4 运动状态推断
- 方法：基于静态背景点云的 ICP 平移估计。
- 利用集装箱堆、地面等固定结构，从静态背景"相对运动"反推 RTG 运动。
- 输出：静止 / 运动 + 方向 (+x 或 -x)。
- 不需要精确速度/里程计。

**ICP 实现关键要点**：
1. **地面点剔除**：必须先做地面分割（RANSAC 或基于高度阈值），仅用非地面静态结构（集装箱边缘、车道边界、路缘石）做 ICP。地面点云在 x 方向平坦退化，缺少几何约束，不加剔除会导致平移估计漂移。
2. **动态目标剔除**：配合检测结果，将落在检测框内的点云从 ICP 匹配中排除，避免行驶中的集卡"拉偏"运动估计。
3. **实现方案**：基于 Open3D 或 PCL 实现，核心步骤为 地面分割 → 动态点剔除 → 静态结构提取 → 帧间平移 ICP → 方向/运动判断。

## 7. 接口协议

### 7.1 ROS1 Topics

| Topic | 消息类型 | 频率 | 说明 |
|---|---|---|---|
| `/rtg_bev/detections` | 自定义 DetectionArray | ≥10 Hz | 3D/BEV 检测结果 |
| `/rtg_bev/tracks` | 自定义 TrackArray | ≥10 Hz | 跟踪目标（第一版内部使用） |
| `/rtg_bev/warnings` | 自定义 WarningArray | ≥10 Hz | 预警等级 + 危险目标列表 |
| `/rtg_bev/ego_motion` | 自定义 EgoMotionState | ≥10 Hz | 运动/静止 + 方向 |
| `/rtg_bev/diagnostics` | diagnostic_msgs/DiagnosticArray | 1 Hz | 系统状态、时延、帧率 |

### 7.2 部署输入

| 来源 | 类型 | 说明 |
|---|---|---|
| 集卡侧相机 | `sensor_msgs/Image` | 2 路, 编码待确认 |
| 禁行侧相机 | `sensor_msgs/Image` | 2 路, 第一版仅录制/可视化 |
| OS1 雷达 | `sensor_msgs/PointCloud2` | 2 路, x,y,z,intensity,timestamp |
| RoboSense 32线雷达 | `sensor_msgs/PointCloud2` | 2 路, 第一版仅录制 |
| 外参 | 标定文件 (YAML) | 雷达-雷达、相机-雷达、车体坐标系 |

## 8. 测试验收

### 8.1 离线评估
- 验证集 ~500 帧。
- 指标：自定义 mAP、NDS、关键目标召回率 (≥98%)、误报率。
- 连续序列评估：跟踪稳定性、预警延迟。

### 8.2 现场验收
- 功能演示/测试用例通过率优先。
- 典型测试用例：
  - 集卡从远处驶近 RTG → 正确检测并预警。
  - 人员进入集卡车道 → 正确检测并预警。
  - RTG 静止时目标进入 → 检测但不预警。
  - 夜间/强光 → 检测效果不显著退化。
  - RTG 双向行驶 → 正确切换预警方向。

### 8.3 性能指标

| 指标 | 目标 | 验证方式 |
|---|---|---|
| 响应延迟 | ≤150 ms | 端到端时间戳差值 |
| 帧率 | ≥10 Hz | 输出 topic 频率 |
| 关键目标召回率 | ≥98% (50m内非遮挡) | 离线评估 + 现场测试用例 |
| 盲区 | ≤1% (防撞有效 ROI 内) | 现场测试点覆盖统计 |

## 10. 风险清单

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 5k 帧标注不足以 fine-tune CenterPoint | 检测效果不达标 | 优先验证预训练权重迁移效果，必要时追加标注 |
| 标定精度不足 | 检测偏移 | 输出标定需求文档，参与标定验收 |
| CenterPoint 在工控机上推理延迟超 150ms | 不满足实时性 | 先测单卡 PyTorch 延迟，必要时 CUDA-CenterPoint (TensorRT) |
| rosbag 数据质量差 (曝光、遮挡、同步) | 训练数据不可用 | 尽早获取和分析 bag，发现问题及时反馈现场 |
| 禁行侧方案未定 | 360° 覆盖不完整 | 第一版以集卡侧为主，禁行侧通过规则/日后补充 |
