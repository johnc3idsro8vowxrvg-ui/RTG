## 2. rosbag 抽取工具 (rosbag_extract)

### 2.1 功能需求

从 ROS1 rosbag 中抽取以下数据:

| 输出 | 格式 | 说明 |
|---|---|---|
| 图像 (前相机) | `front/<timestamp>.jpg` | 原始编码转 JPEG |
| 图像 (后相机) | `rear/<timestamp>.jpg` | 同上 |
| 点云 (前雷达) | `lidar_front/<timestamp>.bin` | float32, N×5 |
| 点云 (后雷达) | `lidar_rear/<timestamp>.bin` | float32, N×5 |
| 时间戳索引 | `timestamps.csv` | 帧对齐后的时间戳表 |
| 标定信息 | `calib.yaml` | 从 /tf 或 /camera_info 提取 |

## 3. 格式转换脚本 (pkl_generator)

### 3.1 功能

将 rosbag_extract 输出转换为 info.pkl:

```python
#!/usr/bin/env python3
"""
generate_rtg_infos.py — 生成 info.pkl (CenterPoint 兼容)

输入:
  output_dir/
    front/       — 前相机图像
    rear/        — 后相机图像
    lidar_front/ — 前雷达点云
    lidar_rear/  — 后雷达点云
    timestamps.csv
    calib.yaml
    annotations.json  — 标注结果

输出:
  rtg_infos_train.pkl
  rtg_infos_val.pkl
"""

def generate_infos(data_dir, ann_file, output_path):
    """生成 info.pkl"""
    infos = []
    
    annotations = load_annotations(ann_file)
    calib = load_calib(os.path.join(data_dir, 'calib.yaml'))
    timestamps = load_timestamps(os.path.join(data_dir, 'timestamps.csv'))
    
    for idx, ts in enumerate(timestamps):
        info = {
            'token': f'rtg_{idx:06d}',
            'timestamp': ts,
            
            # LiDAR
            'lidar_path': {
                'lidar_os1_front': f'lidar_front/{idx:06d}.bin',
                'lidar_os1_rear': f'lidar_rear/{idx:06d}.bin',
            },
            'num_lidar_pts': None,  # 可选
            
            # Cameras (2 views, 命名兼容 nuScenes 惯例)
            'cams': {
                'CAM_FRONT': {
                    'data_path': f'front/{idx:06d}.jpg',
                    'sensor2lidar_rotation': calib['camera_front_to_lidar']['R'],
                    'sensor2lidar_translation': calib['camera_front_to_lidar']['T'],
                    'cam_intrinsic': calib['cameras']['front']['intrinsic'],
                },
                'CAM_BACK': {
                    'data_path': f'rear/{idx:06d}.jpg',
                    'sensor2lidar_rotation': calib['camera_rear_to_lidar']['R'],
                    'sensor2lidar_translation': calib['camera_rear_to_lidar']['T'],
                    'cam_intrinsic': calib['cameras']['rear']['intrinsic'],
                },
            },
            
            # Lidar-to-world transform (以 集卡侧前雷达为世界原点)
            'lidar2world': np.eye(4).tolist(),
            
            # Sweeps (单帧模式为空列表)
            'sweeps': [],
            
            # GT annotations (如果有标注)
            'gt_boxes': ...,
            'gt_names': ...,
        }
        infos.append(info)
    
    # 保存
    import pickle
    with open(output_path, 'wb') as f:
        pickle.dump({'infos': infos, 'metadata': {...}}, f)
```

## 4. 数据采集计划

### 4.1 采集场景

| 场景 | 优先级 | 目标帧数 | 说明 |
|---|---|---|---|
| RTG 正常行驶 + 集卡正常驶入驶出 | 高 | ~3000 | 核心场景 |
| 多集卡同时在场 | 中 | ~300 | 2~3 辆集卡 |
| RTG 双向行驶 | 高 | ~500 | 前后两个方向均有数据 |
| 夜间/低照度 | 高 | ~500 | 有补光设备 |
| 强光/过曝 | 中 | ~300 | 日间直射光 |
| 人员入侵 (禁行侧或集卡侧) | 中 | ~200 | 如无法自然采集可模拟 |
| 静止目标 (停靠集卡/车辆) | 低 | ~200 | 测试静止不预警逻辑 |

### 4.2 采集要求

- 传感器硬同步已实现。
- 采集时记录 RTG 运动状态 (人工标注或同步采集陀螺仪/编码器数据，用于 Ground Truth 运动状态)。
- 每条 rosbag 建议不超过 10 分钟 (控制文件大小和解析效率)。
- 采集时尽量覆盖不同时段 (早/中/晚) 和天气条件。

### 4.3 测量需求

需向现场/甲方获取的几何测量数据:

| 测量项 | 用途 | 精度要求 |
|---|---|---|
| 集卡侧前后支腿坐标 (BEV 系) | 自车 footprint 配置 | ±5 cm |
| 禁行侧前后支腿坐标 (BEV 系) | 自车 footprint 配置 | ±5 cm |
| 大车道宽度 | 场地先验配置 | ±10 cm |
| 集卡车道宽度 | 场地先验配置 | ±10 cm |
| 相机精确安装位置和朝向 | 标定验证 | ±5 cm, ±1° |
| 雷达精确安装位置和朝向 | 标定验证 | ±5 cm, ±1° |

## 5. 数据质量检查

### 5.1 rosbag 质量报告

```python
def generate_quality_report(bag_path):
    """生成数据质量报告"""
    report = {
        'duration': ...,           # 总时长
        'total_frames': ...,       # 总帧数
        'sync_loss_rate': ...,     # 同步丢失率
        'camera_stats': {          # 各相机统计
            'front': {'fps': ..., 'drop_rate': ..., 'overexposure_ratio': ...},
            'rear':  {...},
        },
        'lidar_stats': {           # 各雷达统计
            'front': {'fps': ..., 'drop_rate': ..., 'points_per_frame': ...},
            'rear':  {...},
        },
        'timestamp_jitter': ...,   # 时间戳抖动
    }
    return report
```

### 5.2 检查标准

| 检查项 | 合格标准 |
|---|---|
| 同步丢失率 | <5% |
| 相机帧率 | 按标称值 ±10% |
| 雷达帧率 | 10 Hz ±10% (OS1 / RoboSense) |
| 单帧点云数 (OS1) | >50k (正常场景) |
| 过曝/过暗图像比例 | <10% |
| 时间戳单调性 | 无回跳 |
