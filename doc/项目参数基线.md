# RTG BEV 项目参数基线

> 本文档记录项目当前权威口径。其他文档如有冲突，以本文档为准。

## 1. 坐标系

| 项目 | 基线 |
|---|---|
| 坐标系 | 固定车体系，不随 RTG 行驶方向翻转 |
| 原点 | L1（集卡侧前支腿 OS1 128 线雷达）地面投影 |
| +x | 大车道方向，C1 朝向 |
| +y | 跨距方向，禁行侧 → 集卡车道侧 |
| +z | 竖直向上 |
| BEV 范围 | x ∈ [-60, 60] m, y ∈ [-25, 25] m, z ∈ [-3, 8] m |
| 相邻箱区镜像轴 | y = +4.25 |
| 镜像公式 | y' = 8.5 - y |

## 2. 传感器与 V1 范围

| 范围 | 说明 |
|---|---|
| V1 推理闭环 | 集卡侧 L1/L2 两个 OS1 128 线雷达，点云拼接后进入 CenterPoint |
| V1 录制/可视化 | C1~C4 四相机 + L1~L4 四雷达，共 8 传感器 |
| 后续扩展 | 禁行侧感知、相机融合、地面投影与视觉场地先验 |

| 传感器 | 型号/规格 | 频率 | V1 角色 |
|---|---|---|---|
| C1/C2 | 海康工业相机，竖屏 1080×1920，bgr8 | 10 Hz | 录制/可视化；融合扩展 |
| C3/C4 | 海康工业相机，竖屏 1080×1920，bgr8 | 10 Hz | 录制/可视化；禁行侧扩展 |
| L1/L2 | Ouster OS1 128 线，约 262k pts/帧 | 10 Hz | V1 主推理输入 |
| L3/L4 | RoboSense 32 线，约 16k~25k pts/帧 | 10 Hz | 录制/可视化；禁行侧扩展 |

## 3. 模型路线

| 项目 | 基线 |
|---|---|
| 主模型 | CenterPoint LiDAR-only |
| 输入 | 集卡侧 L1/L2 拼接单帧点云 |
| 输出类别 | person / truck / car / other_obstacle |
| 跟踪 | V1 需要短时跟踪模块，提供稳定 track_id 供预警关联 |
| 相机融合 | 不属于 V1 推理闭环，作为后续扩展 |

## 4. 标定要求

| 阶段 | 必须项 |
|---|---|
| V1 LiDAR-only 闭环 | L1/L2 雷达拼接外参、传感器到车体坐标、雷达水平约束、时间同步验证 |
| 相机/融合扩展 | 相机内参、相机畸变、相机到雷达外参、相机到地面投影验证 |

## 5. 预警策略

V1 预警由以下信息驱动：

1. 距离阈值：目标框中心到最近 RTG 支腿的 BEV 平面距离。
2. 运动方向：静止、运动 +x、运动 -x、未知。
3. 可选场地先验：车道/区域信息仅作为降权或保守策略，不作为固定 ROI 硬依赖。

危险等级单帧触发；警告/提示连续 3 帧确认；解除延迟 3 帧。

## 6. 性能统计口径

性能验收需分阶段记录 P50/P95：

| 阶段 | 统计起止点 |
|---|---|
| 预处理 | ROS 消息接收 → 点云过滤/格式整理完成 |
| 点云拼接 | L1/L2 点云读入 → 统一 BEV 坐标系点云输出 |
| 模型推理 | CenterPoint 输入张量就绪 → 检测框输出 |
| 后处理 | 检测框输出 → 跟踪、距离计算、预警计算完成 |
| ROS 发布 | 发布对象生成 → topic publish 调用完成 |
| 端到端 | 传感器时间戳 → 对应输出 topic 时间戳 |

输出帧率目标为 ≥10 Hz。最终延迟阈值应在工控机基准测试后冻结。

## 7. 评估误报口径

检测误报不使用 FPR 作为主指标。优先统计：

- FP/frame
- FP/min
- 按距离段统计 FP 数量：[0,10] m、(10,25] m、(25,50] m
