# RTG BEV 标注规范

> 本文档定义 RTG 大车防撞 BEV 检测数据集的标注标准、工具配置和质量检查流程。
> 标注规模: ~5000 帧 (4500 训练 + 500 验证)，仅标注集卡车道侧。

---

## 1. 标注范围

### 1.1 传感器覆盖
仅标注集卡车道侧传感器覆盖区域：
- 2× OS1 128 线激光雷达（集卡侧前后支腿）
- 2× 海康工业相机（集卡侧前后支腿）

禁行侧数据不参与第一版标注。

### 1.2 空间范围
标注有效区域：BEV 坐标系下 x ∈ [-60, 60]m, y ∈ [-25, 25]m, z ∈ [-3, 8]m。

### 1.3 标注帧分配

| 类型 | 数量 | 说明 |
|---|---|---|
| 检测关键帧 | ~4000 | 覆盖正常作业场景（日间/夜间/强光混合） |
| 困难帧 | ~500 | 夜间、低照度、强光、部分遮挡等困难场景 |
| 连续序列 | ~500 | 5~10 个片段，每片段 50~100 帧，覆盖集卡完整驶过过程 |

## 2. 类别定义

### 2.1 类别列表

| ID | 类别名 | 定义 | 标注说明 |
|---|---|---|---|
| 0 | `person` | 人员（行人、作业人员） | 标注整个人体 3D 框 |
| 1 | `truck` | 集卡（整体，含牵引车+挂车+载箱） | 整体标注，不拆分子部件 |
| 2 | `car` | 乘用车/小型车辆 | 标准 3D 框 |
| 3 | `other_obstacle` | 其他近地障碍物 | 含: 邻车 RTG、AGV/IGV、叉车、锥桶、路障、临时设施、未知障碍物 |

### 2.2 不标注的对象
- 高空结构: 吊具、大梁、悬挂物（仅关注地面/近地目标）
- 本 RTG 自车结构: 支腿、连接梁、设备罩（通过自车 footprint mask 排除）
- 集装箱堆（静态，不作为障碍物）
- 地面标志线、路面污渍、井盖

### 2.3 标注优先级
集卡 (truck) > 人员 (person) > 乘用车 (car) > 其他障碍物 (other_obstacle)

同一帧中优先确保集卡和人员的标注完整准确。

---

## 3. 3D 框标注规范

### 3.1 标注坐标系
- 坐标系: BEV 坐标系（原点 = 集卡侧 OS1 雷达地面投影）
- x 轴: 大车道方向
- y 轴: 跨距方向（从集卡侧指向禁行侧为 +y）
- z 轴: 竖直向上

### 3.2 框参数定义

每个 3D 框包含 7 个参数: `(x, y, z, w, l, h, yaw)`

| 参数 | 含义 | 单位 |
|---|---|---|
| x, y, z | 框中心点坐标 | 米 |
| w | 宽度 (y 方向) | 米 |
| l | 长度 (x 方向) | 米 |
| h | 高度 (z 方向) | 米 |
| yaw | 朝向角 (绕 z 轴旋转，弧度) | rad |

### 3.3 各类别标注细则

#### person（人员）
```
- 框尺寸: 约 0.5~0.8m × 0.5~0.8m × 1.6~1.9m (w × l × h)
- 中心点: 人体几何中心
- yaw: 面向方向，如难以判断可设为 0
- 标注条件: 至少 10 个 LiDAR 点落在人体上
- 远距离 (≥40m): 如果仅少量点云但明显是人，仍需标注
```

#### truck（集卡）
```
- 框尺寸: 约 2.5~3m × 12~18m × 3~4.5m (w × l × h)，依实际尺寸
- 中心点: 集卡整体几何中心（含牵引车+挂车+载箱）
- yaw: 集卡行驶方向，RTG 场景基本为 0° 或 180°（沿车道）
- 不拆分: 牵引车、挂车、集装箱作为整体标注一个框
- 如果在载箱上方货物超出，框高度需覆盖货物
- 标注条件: 集卡至少部分进入 50m 检测范围
```

#### car（乘用车）
```
- 框尺寸: 约 1.8~2m × 4~5m × 1.4~1.8m (w × l × h)
- 中心点: 车辆几何中心
- yaw: 车辆行驶方向
```

#### other_obstacle（其他障碍物）
```
- 框尺寸: 按实际近地障碍物轮廓
- 包含: 邻车 RTG（作为整体大框）、AGV、叉车、锥桶、路障
- 对于锥桶等小目标: 标注尽量紧密的 3D 框
- 对于邻车 RTG: 可不追求精确框，近地部分标注即可
```

### 3.4 3D 框贴合度要求
- 框应紧密贴合目标在点云中可见的点。
- 被自车结构、箱堆部分遮挡的目标：仅根据可见部分的点云估计完整框。
- 如果遮挡超过 70% 且无法推断完整框大小 → 不标注。

## 4. 标注流程

### 4.1 标注工具配置

推荐工具: **CVAT** (支持 3D 点云标注) 或 **LabelCloud**

工具配置要求:
- 导入: 点云 (.bin) + 图像 (.jpg) + 标定文件 (外参 YAML)
- 视图: 3D 点云视图 (主标注视图) + BEV 鸟瞰视图 (辅助) + 图像视图 (辅助质检)
- 输出: 自定义 JSON/PKL 格式，包含 3D 框参数 + 类别

### 4.2 标注步骤

1. **数据准备**
   - 从 rosbag 抽取点云和图像
   - 生成 nuScenes 兼容的 info.pkl 骨架
   - 配置标定参数（相机内外参、雷达外参）

2. **预标注（可选）**
   - 使用 NuScenes pretrained checkpoint 做初步推理
   - 人工修正预标注结果

3. **人工标注**
   - 在 3D 点云视图中逐帧标注
   - 优先标注 truck 和 person
   - BEV 视图辅助调整 x/y/朝向
   - 图像视图辅助确认类别

4. **质检**
   - 逐帧检查框的贴合度
   - 检查漏标和误标
   - 交叉验证（标注员 A 标注，标注员 B 抽检）

5. **格式转换**
   - 标注结果 → info.pkl

### 4.3 标注效率估算

| 类别 | 每框标注时间 (估算) | 每帧平均框数 | 每帧标注时间 |
|---|---|---|---|
| person | ~20s | 0~2 | ~20s |
| truck | ~30s | 0~3 | ~45s |
| car | ~20s | 0~1 | ~10s |
| other | ~25s | 0~3 | ~30s |
| **合计** | | | **~2 min/帧** |

5000 帧 × 2 min ≈ **~170 标注工时** (不含质检)

## 5. 质量检查标准

### 5.1 质检抽样
- 全部帧: 逐帧浏览，检查明显漏标/误标
- 详细检查: 随机抽取 10% (500 帧)，检查框贴合度和类别正确性
- 连续序列: 100% 详细检查（对跟踪评估至关重要）

### 5.2 合格标准

| 检查项 | 标准 |
|---|---|
| 框贴合度 | ≥90% 的框与点云贴合度良好（IoU 人工评估 ≥0.8） |
| 漏标率 | 50m 内非遮挡目标漏标率 <5% |
| 误标率 | 非目标被标注为目标 <2% |
| 类别正确率 | ≥95% |
| yaw 正确率 | truck 和 car 的 yaw 误差 <15° |

### 5.3 常见标注错误
- 把集装箱边缘误标为车辆
- 把地面反光/污渍误标为障碍物
- 自车支腿或设备罩被标为目标
- 载箱集卡只标了底盘没标载箱高度
- 忽略被部分遮挡的目标

## 6. 输出格式

### 6.1 标注文件格式

每帧一个标注 JSON（或合并为一个标注文件）:

```json
{
  "frame_idx": 0,
  "timestamp": 1700000000.123456,
  "boxes": [
    {
      "class": "truck",
      "x": 25.3, "y": 3.2, "z": 1.8,
      "w": 2.6, "l": 15.2, "h": 4.1,
      "yaw": 0.02,
      "occluded": 0,
      "truncated": 0
    }
  ]
}
```

### 6.2 转换脚本

标注 JSON → info.pkl 格式的转换脚本在数据方案文档中详细描述。
