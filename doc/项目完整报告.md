## 1. 执行摘要

RTG BEV 大车防撞感知系统面向港口轮胎式集装箱门式起重机（RTG）的大车行走安全场景，目标是在 BEV 坐标系下输出标准化 3D/BEV 障碍物检测结果、短时跟踪结果、运动状态估计和分级碰撞风险预警。

项目当前采用 CenterPoint LiDAR-only 技术路线。V1 推理闭环以集卡侧 L1/L2 两个 Ouster OS1 128 线雷达为主输入，通过外参拼接形成统一单帧点云，再进入 CenterPoint 检测。4 相机 + 4 雷达的 8 传感器数据在 V1 中用于录制、回放和可视化；禁行侧检测和相机融合保留为后续扩展。

当前项目状态为 Phase 2 已完成：真实测试数据完成提取与参数确认，CenterPoint 迁移完成，数据管线和 RViz 全传感器可视化已跑通。下一阶段的关键前置条件是 V1 必需标定项、5000 帧训练数据采集和 3D 框标注。

## 2. 项目范围与目标

### 2.1 建设目标

系统建设目标包括：

1. 在 RTG 大车道方向 50m 范围内稳定检测人员、集卡、车辆和其他近地障碍物。
2. 输出统一 BEV 坐标系下的 3D/BEV 检测框。
3. 基于检测、跟踪、距离阈值和 RTG 运动方向输出分级预警。
4. 提供 ROS1 标准化接口，支持在线推理、离线 rosbag 回放、日志和诊断。
5. 支持现场部署、验收测试、复盘分析和后续禁行侧/相机融合扩展。

### 2.2 项目边界

本项目是纯感知系统，不接入 RTG 控制系统，不直接执行制动或控制动作。系统运行时不依赖 RTG 轮速、编码器、IMU、GPS 等控制状态输入，RTG 的运动/静止和方向需要通过传感器数据推断。评估和验收阶段可额外记录人工或外部运动真值，用于验证运动估计准确性。

V1 范围聚焦集卡侧大车防撞闭环。禁行侧和相机融合数据保留，但不作为 V1 主推理闭环的硬依赖。

### 2.3 V1 边界

| 范围 | 内容 |
|---|---|
| V1 推理闭环 | 集卡侧 L1/L2 两个 OS1 128 线雷达，点云拼接后进入 CenterPoint |
| V1 录制/可视化 | C1~C4 四相机 + L1~L4 四雷达，共 8 传感器 |
| 后续扩展 | 禁行侧感知、相机融合、地面投影与视觉场地先验 |

## 3. 场景、坐标系与几何基线

### 3.1 场景结构

目标场景为标准化矩形港口堆场。大车道、集卡车道和集装箱列均沿 RTG 大车运行方向平行布置。RTG 跨距约 23.5m，前后支腿轮组间距约 12.0m，跨距范围内包含六条集装箱道和一条集卡车道。地面黄线和固定箱区结构可作为几何先验。

### 3.2 BEV 坐标系

| 项目 | 定义 |
|---|---|
| 坐标系 | 固定车体系，不随 RTG 行驶方向翻转 |
| 原点 | L1（集卡侧前支腿 OS1 128 线雷达）地面投影 |
| +x | 大车道方向，C1 朝向 |
| +y | 跨距方向，禁行侧 → 集卡车道侧 |
| +z | 竖直向上 |
| BEV 范围 | x ∈ [-60, 60] m, y ∈ [-25, 25] m, z ∈ [-3, 8] m |

相邻箱区车道布局关于中间车道中心线 `y=+4.25` 镜像对称，镜像公式为 `y'=8.5-y`。该结论用于解释相邻箱区的语义不变性和传感器位置关系。

### 3.3 RTG 几何参数

| 参数 | 值 | 说明 |
|---|---|---|
| 跨距 | 23.5m | 集卡侧到禁行侧大车道中心距离 |
| 轮组间距 | 12.0m | 前后支腿 x 方向距离 |
| 支腿宽度 | 1.0m | y 方向 |
| 支腿长度 | 1.0m | x 方向 |
| 相机高度 | 约 4.0m | 支腿上方安装 |
| 雷达高度 | 约 1.5m | 支腿下方安装 |

自车 footprint 分为集卡侧和禁行侧两个矩形区域，用于剔除自车结构点云、辅助 BEV 可视化和预警距离计算。

### 4.1 传感器配置

### 4.2 传感器位置

| 传感器 | BEV 坐标 (x, y, z) | 侧别 | 支腿 |
|---|---|---|---|
| C1 | (0, 0, 4.0) | 集卡侧 | 前 |
| C2 | (-12.0, 0, 4.0) | 集卡侧 | 后 |

### 4.3 FOV 与能力特点

OS1 128 线雷达水平 360° 覆盖，前方约 180° 可用，垂直 FOV 约 42.4°，单帧点数约 262k，主用于集卡侧检测。RoboSense 32 线雷达旋转 90° 安装后沿大车道方向形成约 70° 水平覆盖，点云较稀疏，V1 用于录制/可视化和后续禁行侧扩展。海康相机为竖屏安装，俯视图锥角约 46°，后续可用于视觉确认、地面投影和相机融合。

## 5. 算法与系统架构

### 5.1 总体架构

系统按传感器层、感知层、后处理层和输出层组织：

1. 传感器层接收 L1/L2 主雷达点云，并保留 8 传感器录制和可视化能力。
2. 感知层使用 CenterPoint LiDAR-only 主模型，输入为集卡侧双雷达拼接单帧点云。
3. 后处理层包含短时跟踪、运动状态估计、自车 footprint 过滤和分级预警。
4. 输出层通过 ROS1 发布 detections、tracks、warnings、ego_motion 和 diagnostics。

### 5.2 CenterPoint 配置

| 配置项 | 当前方案 |
|---|---|
| 框架 | CenterPoint / det3d |
| 检测器 | VoxelNet |
| Backbone | SpMiddleResNetFHD |
| Neck | RPN/FPN |
| Head | CenterHead |
| 输入雷达 | L1/L2 两个 128 线雷达拼接 |
| 点云范围 | [-60,-25,-3, 60,25,8] |
| 体素尺寸 | [0.075, 0.075, 0.2] |
| 类别数 | 4 类 |
| 时序策略 | 单帧，不聚合 sweeps |

### 5.3 检测类别

| ID | 类别 | 说明 |
|---|---|---|
| 0 | person | 人员、作业人员 |
| 1 | truck | 集卡整体，含牵引车、挂车和载箱 |
| 2 | car | 乘用车或小型车辆 |
| 3 | other_obstacle | 邻车 RTG、AGV/IGV、叉车、锥桶、路障等 |

### 5.4 后处理模块

短时跟踪模块用于提供跨帧稳定 `track_id`，V1 预警需要依赖该 ID 关联目标。运动估计采用静态背景点云 ICP 方法，结合地面点剔除和动态目标剔除，推断 RTG 静止、运动 +x、运动 -x 或未知状态。自车 footprint 过滤用于移除支腿和连接梁等自车结构点。

## 6. 预警策略

V1 预警不依赖相机投影固定 ROI，而由距离阈值、运动方向和可选场地先验共同驱动。

### 6.1 预警等级

| 等级 | 含义 | 触发方式 |
|---|---|---|
| 0 | 无风险 | 不触发 |
| 1 | 提示 | 连续 3 帧确认 |
| 2 | 警告 | 连续 3 帧确认 |
| 3 | 危险 | 单帧即时触发 |

### 6.2 初始阈值

| 类别 | danger | warning | info |
|---|---|---|---|
| person | 8m | 15m | 25m |
| truck | 12m | 25m | 40m |
| car | 10m | 20m | 35m |
| other_obstacle | 10m | 20m | 35m |

距离定义为目标框中心到最近 RTG 支腿的 BEV 平面距离。RTG 静止时默认只输出检测，不主动预警；运动方向前方目标保持原始等级，后方或侧向目标可按场地先验降权。

## 8. 标注方案

### 8.1 标注范围

V1 训练标注以集卡侧 L1/L2 点云 3D 框为主。C1/C2 相机仅用于类别确认、质检辅助和后续 3D→2D 投影扩展，不作为 V1 CenterPoint LiDAR-only 主模型输入。禁行侧数据第一版不参与标注，但保留用于后续扩展。

标注空间范围为 x ∈ [-60,60]m、y ∈ [-25,25]m、z ∈ [-3,8]m。标注对象为 person、truck、car、other_obstacle 四类。

### 8.2 标注规模

| 类型 | 数量 | 说明 |
|---|---|---|
| 检测关键帧 | 约 4000 | 正常作业场景 |
| 困难帧 | 约 500 | 夜间、低照度、强光、遮挡等 |
| 连续序列 | 约 500 | 5~10 个片段，每片段 50~100 帧 |

按估算，每帧约 2 分钟，5000 帧约 170 标注工时，不含质检。

### 8.3 标注质量标准

| 检查项 | 标准 |
|---|---|
| 框贴合度 | ≥90% 的框与点云贴合良好 |
| 漏标率 | 50m 内非遮挡目标漏标率 <5% |
| 误标率 | 非目标被标注为目标 <2% |
| 类别正确率 | ≥95% |
| yaw 正确率 | truck 和 car 的 yaw 误差 <15° |

## 9. 标定方案

### 9.1 标定分层

| 阶段 | 必须项 |
|---|---|
| V1 LiDAR-only 闭环 | L1/L2 雷达拼接外参、传感器到车体坐标、雷达水平约束、时间同步验证 |
| 相机/融合扩展 | 相机内参、相机畸变、相机到雷达外参、相机到地面投影验证 |

### 9.2 关键精度要求

| 项目 | 阶段 | 精度要求 |
|---|---|---|
| 雷达到雷达外参 | V1 必须 | 旋转 ±0.3°，平移 ±2cm |
| 传感器到车体坐标 | V1 必须 | 旋转 ±1°，平移 ±5cm |
| 雷达扫描平面水平 | V1 必须 | 与地面夹角 <1° |
| 雷达间同步 | V1 必须 | <5ms |
| 相机内参 | 融合扩展必须 | fx/fy ±2%，cx/cy ±5px，重投影 RMS <0.5px |
| 相机到雷达外参 | 融合扩展必须 | 旋转 ±0.3°，平移 ±2cm |
| 相机间同步 | 融合扩展必须 | <5ms |
| 相机到雷达同步 | 融合扩展必须 | <50ms |

### 9.3 交付物

V1 必须交付 L1/L2 雷达拼接外参、传感器到车体外参、雷达水平面校准报告、时间同步验证报告、自车支腿地面投影坐标和标定精度报告。相机内参和畸变、相机到雷达外参、投影可视化、地面标记投影验证报告属于相机/融合扩展交付物，可与 V1 标定分批验收。

## 12. 测试与验收

### 12.1 验收优先级

验收优先级为：功能演示通过、现场测试用例通过率、离线评估指标。离线评估作为技术参考，现场功能闭环和安全行为更关键。

### 12.2 离线指标

| 指标 | 目标/口径 |
|---|---|
| 关键目标召回率 | ≥98%，50m 内，IoU≥0.3 |
| Precision | ≥80% |
| FP/frame | 每帧平均误检数，越低越好 |
| FP/min | 每分钟平均误检数，越低越好 |
| 分距离段 FP | [0,10]m、(10,25]m、(25,50]m |
| ATE | ≤0.5m@50m |
| ASE | ≤10% |
| 跟踪 ID switch | <5% |
| 跟踪连续性 | >90% |
| 预警延迟 | <500ms |

除关键目标召回率和 Precision 外，FP/frame、FP/min、分距离段 FP、预警延迟等指标当前主要作为统计口径；最终通过阈值需结合现场数据和甲方验收要求冻结。

### 12.3 性能统计

性能验收需分别统计 P50/P95：

| 阶段 | 起止点 |
|---|---|
| 预处理 | ROS 消息接收 → 点云过滤/格式整理完成 |
| 点云拼接 | L1/L2 点云读入 → 统一 BEV 点云输出 |
| 模型推理 | CenterPoint 输入就绪 → 检测框输出 |
| 后处理 | 检测框输出 → 跟踪/距离/预警完成 |
| ROS 发布 | 发布对象生成 → topic publish 调用完成 |
| 端到端 | 传感器时间戳 → 输出 topic 时间戳 |

输出帧率目标为 ≥10Hz。各阶段 P50/P95 延迟当前为统计口径，最终延迟预算需在工控机基准测试后冻结；冻结前不应把单一固定毫秒值作为正式验收阈值。

### 12.4 现场测试用例

现场验收包括 8 类核心用例：集卡前方驶近、集卡后方驶近、人员进入集卡车道、RTG 静止、多目标同时存在、运动状态推断、静止目标误报抑制和系统性能测试。正式版需 TC-01~TC-08 全部通过，并满足性能指标。

## 14. 当前主要风险

| 风险 | 影响 | 应对 |
|---|---|---|
| V1 必需标定未冻结 | 双雷达拼接、车体坐标和水平约束不稳定 | 优先完成 C3/C4/C7/T1 标定验收 |
| 5000 帧训练数据未采集 | 无法训练 RTG 场景检测模型 | 标定完成后立即组织多场景采集 |
| 点云密度高导致工控机延迟超预算 | 实时性风险 | 推理阶段评估 voxel_size、降采样、TensorRT |
| 域差距导致零样本检测为 0 | 检测效果不达标 | 依赖 RTG 标注数据 fine-tune |
| 禁行侧方案延后 | 覆盖范围不完整 | V1 明确以集卡侧为主，禁行侧保留数据与扩展路线 |
| 相机融合依赖额外标定 | 融合扩展被标定质量制约 | 将相机内参/外参和地面投影列为融合扩展必须项 |

## 15. 下一步建议

1. 冻结 V1 LiDAR-only 必需标定：L1/L2 拼接外参、传感器到车体坐标、雷达水平约束和时间同步。
2. 使用冻结标定重新验证 L1/L2 点云拼接、self-footprint 过滤和 CenterPoint DataLoader。
3. 采集正式 5000 帧训练数据，确保覆盖正常作业、夜间、强光、人员入侵、多目标、双向行驶和静止目标场景。
4. 启动标注，优先保证 truck/person 质量，连续序列做 100% 详细质检。
5. 完成首轮 CenterPoint fine-tune，重点跟踪 50m 内关键目标召回率和距离分段误检。
6. 集成跟踪、运动估计和预警模块，跑通 rosbag 全链路回放。
7. 在工控机上记录分阶段 P50/P95 延迟，并据此冻结最终性能预算。
8. 准备 TC-01~TC-08 现场验收脚本、记录模板和复盘流程。
