## 1. 项目概述

### 1.1 核心目标

为轮胎式集装箱门式起重机（RTG，轮胎吊）构建面向大车防撞的 BEV 感知系统。系统输出：

1. 标准化 3D/BEV 障碍物检测结果
2. 分级碰撞风险/防撞预警信息

| 项目 | 说明 |
|---|---|
| 检测类别 | person / truck / car / other_obstacle（4 类） |
| BEV 范围 | x∈[-60,60]m, y∈[-25,25]m, z∈[-3,8]m |
| 感知距离 | ≥50m（沿大车道方向） |
| 延迟统计 | 预处理/拼接/推理/后处理/ROS发布/端到端分别统计 P50/P95 |
| 输出帧率 | ≥10Hz |
| 通信中间件 | ROS1 |
| 训练平台 | 远程 A800 GPU 服务器 |
| 开发平台 | Ubuntu 20.04 + RTX 3060 (12GB) |
| 部署平台 | 工控机（GPU 型号待定） |

### 1.2 项目边界

- 纯感知项目，不接入 RTG 控制系统
- 不获取 RTG 运动控制状态（轮速、编码器、IMU、GPS 等），需从传感器数据推断运动状态
- 大车静止时不主动预警，仅检测
- 最终交付为产品级系统（稳定在线、标准接口、可视化、日志、异常恢复）

### 1.3 验收方式

- 甲方验收：功能演示/现场测试用例通过率优先，同时兼顾离线测试指标
- mAP/NDS 使用项目自定义类别和距离范围，不绑定 NuScenes 官方评测
- 盲区仅统计防撞有效 ROI 内，不按 360° 全域面积

> 完整验收标准与测试用例详见 **[测试验收方案](./测试验收方案.md)**。

## 2. 检测类别与防撞对象

| ID | 类别 | 说明 |
|---|---|---|
| 0 | `person` | 人员（行人、作业人员） |
| 1 | `truck` | 集卡（整体，含牵引车+挂车+载箱） |
| 2 | `car` | 乘用车/小型车辆 |
| 3 | `other_obstacle` | 其他近地障碍物（邻车RTG、AGV/IGV、叉车、锥桶等） |

**设计决策**：第一阶段不分细类。防撞重点是人和集卡。AGV/IGV、叉车、锥桶等归入 `other_obstacle` 兜底。邻车 RTG 不设专门类别（结构复杂、样本少），归入 `other_obstacle`。集卡不拆牵引车/挂车/载箱，从 BEV 占据角度标注整体。

本项目只关注地面/近地障碍物。不关注高处吊具、大梁、悬挂物。

> 详细标注规则、3D框尺寸规范、质检标准详见 **[标注规范](./标注规范.md)**。

## 3. 场景与几何空间

> 完整场景布局、BEV坐标系定义、传感器安装坐标、FOV覆盖、镜像对称关系、集装箱尺寸详见 **[几何空间与传感器FOV参考手册](./几何空间与传感器FOV参考手册.md)**（几何空间的唯一权威来源）。

### 3.1 堆场结构

- 标准化矩形堆场。大车道、集卡车道、集装箱列均平行，沿大车道方向（x 轴）
- RTG 跨距 ~23.5m（6+1 布置：6 列箱 + 1 条集卡道）
- RTG 四腿轮组间距 ~12.0m
- 每侧前后支腿通过连接梁连成一体
- 地面有稳定车道黄线，可作为几何先验

### 3.2 BEV 坐标系

```
原点: L1（集卡侧前支腿 OS1 128线雷达）在地面的垂直投影
+x:   大车道方向（C1 朝向），RTG 可沿 ±x 双向行驶
+y:   跨距方向，禁行侧 → 集卡侧（与 ROS 惯例一致）
+z:   竖直向上

固定车体系：不随行驶方向翻转
使用 +x/-x 代替"前/后"（因为前/后随行驶方向变化）
使用 集卡侧(y≈0)/禁行侧(y≈-23.5)代替"左/右"
```

### 3.3 镜像对称关系

两个相邻箱区的车道布局关于**中间车道中心线 y=+4.25** 呈**轴对称（镜像对称）**。
镜像公式：`y' = 8.5 - y`，x 不变。两 RTG 均朝 +x 方向。

核心推论：**无论 RTG 在哪个箱区，同编号相机拍到的画面语义结构完全相同**。
C1 画面从左到右始终为：相邻箱区 → 中间车道 → 自车大车道 → 集卡车道 → 自车集装箱道。

### 3.4 RTG 自车 Footprint

RTG 地面上仅有 4 条支腿。拆分为两个独立矩形：

| 区域 | x 范围 | y 范围 | 说明 |
|---|---|---|---|
| 集卡侧 | [-12.5, 0.5]m | [-0.5, 0.5]m | 前后支腿 + 连接梁，宽 1m |
| 禁行侧 | [-12.5, 0.5]m | [-24.0, -23.0]m | 跨距远端，宽 1m |

两个 footprint 之间为正常检测空间。

## 4. 传感器配置

> 完整参数、FOV 覆盖、安装细节详见 **[几何空间与传感器FOV参考手册](./几何空间与传感器FOV参考手册.md)** 第 3、7 节。

| 侧 | 支腿 | 相机 (海康, ~4m高) | 雷达 (~1.5m高) | 第一版角色 |
|---|---|---|---|---|
| 集卡侧 | 前 (x=0) | C1 | L1: OS1 128线, 262k pts, 10Hz, 含ring | **主力**（训练+推理） |
| 集卡侧 | 后 (x=-12.0) | C2 | L2: OS1 128线, 262k pts, 10Hz, 含ring | **主力** |
| 禁行侧 | 前 (x=0) | C3 | L3: RoboSense 32线, ~17k pts, 10Hz, 旋转90° | 录制延后 |
| 禁行侧 | 后 (x=-12.0) | C4 | L4: RoboSense 32线, ~18k pts, 10Hz, 旋转90° | 录制延后 |

- 相机：1080×1920 竖屏，bgr8，俯视图锥角 46°（沿 y 跨距），87.6°（沿 x 纵深+高度）
- 相机均沿 ±x 安装（C1/C3 朝 +x，C2/C4 朝 -x），斜向下约 4m 高
- OS1 128线：360° 水平，42.4° 垂直，前方 180° 可用
- 禁行侧 32 线：旋转 90° 安装（垂直 360°，水平 ~70°）
- 四相机之间基本无共视，四雷达硬同步

## 5. 模型架构

> 架构总览、数据流详见 **[总体技术方案](./总体技术方案.md)**。

### 5.1 总体架构

基于 CenterPoint (CVPR 2021)，LiDAR-only 单分支：

```
雷达分支:  Voxelization → SpMiddleResNetFHD (SparseConv3D) → RPN (FPN)
检测头:    CenterHead (Heatmap + Regression) → 3D Boxes
```

### 5.2 关键配置项

| 配置项 | 值 (RTG 场景) |
|---|---|
| 激光雷达 | 2× 128线拼接 (外参拼接为统一单帧点云) |
| point_cloud_range | [-60,-25,-3, 60,25,8] |
| voxel_size | [0.075, 0.075, 0.2] |
| 类别数 | 4 |
| sweeps 聚合 | 单帧 (RTG 低速 + 128线密度足够) |
| max_voxel_num | [120000, 160000] |

### 5.3 预训练权重策略

- 加载 NuScenes pretrained CenterPoint checkpoint (backbone + neck + head)
- 整体 fine-tune RTG 数据集

### 5.4 第一版模型边界

- **集卡侧主模型**：CenterPoint LiDAR-only，2 雷达拼接
- **禁行侧**：第一版不处理，数据保留用于后续
- **纯 LiDAR 模式**：CenterPoint 本身就是 LiDAR-only，无需额外分支

## 6. 预警逻辑

> 完整预警参数、ROI配置、热更新接口详见 **[配置说明](./配置说明.md)**。运动估计实现详见 **[总体技术方案](./总体技术方案.md)** 6.4 节。

### 6.1 分级预警

| 等级 | 含义 | 触发方式 |
|---|---|---|
| 0 | 无风险 | — |
| 1 | 提示/关注 | 连续 3 帧确认 |
| 2 | 警告 | 连续 3 帧确认 |
| 3 | 危险 | 单帧即时触发 |

### 6.2 初版距离阈值

| 类别 | 危险 (单帧) | 警告 | 提示 |
|---|---|---|---|
| 人员 | 0~8m | 8~15m | 15~25m |
| 集卡/车辆 | 0~12m | 12~25m | 25~40m |
| 大型/未知障碍物 | 0~10m | 10~20m | 20~35m |

附加规则：仅对大车道核心 ROI 触发高等级预警；横向入侵区可降低一级；RTG 静止时不预警；所有阈值配置化。

### 6.3 运动状态推断

基于静态背景点云 ICP 平移估计。利用集装箱堆、地面等固定结构反推 RTG 运动。输出：静止 / 运动 + 方向（+x 或 -x）。不需要精确速度/里程计。

核心要点：必须先做地面点剔除 + 动态目标剔除，否则平移估计漂移。

## 12. 关键风险

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 标定精度不足（相机-雷达 2.5m 垂直偏移） | 融合效果差，检测偏移 | C2 精度已收紧至 ±0.3°/±2cm；新增 C6/C7 验证项 |
| 5k 帧标注不足以 fine-tune 融合层 | 检测效果不达标 | 建议先标 2000 帧做预实验；必要时冻结 backbone 仅训练融合层 |
| 2 相机无共视 | 每 BEV 位置仅 1 个相机信息源 | 如效果不及预期，考虑时序传播模块（~200 行） |
| CenterPoint 工控机推理超出冻结预算 | 不满足实时性 | CUDA-CenterPoint (TensorRT 加速)；按需调整 voxel_size |
| 相机竖屏 1080×1920（计划横屏） | 预训练 backbone 输入分布不同 | 先保持原生分辨率训练；必要时旋转 90° 后训练 |
| 点云 262k pts（5x 预期） | 推理延迟和显存压力 | 训练全量，推理降采样；调大 voxel_size |
