# RTG BEV 配置说明

> 本文档定义 RTG 大车防撞 BEV 感知系统的完整配置体系，包括所有配置文件的格式、加载方式、消费关系和热更新策略。

---

## 1. 配置文件总览

```
config/
├── calib.yaml          # 标定参数 (标定团队提供)
├── geometry.yaml       # RTG 几何与车道配置 (现场测量 + 图像标注)
├── warning.yaml        # 预警参数 (现场调试)
├── system.yaml         # 系统/部署配置 (部署时配置)
└── model/
    └── rtg_centerpoint_voxelnet.py  # 模型配置 (算法开发)
```

### 1.1 配置文件关系

```
calib.yaml ─────────────┐
                         ├──→ 数据预处理 (投影/点云拼接)
geometry.yaml ──────────┤
                         │
model config (.py) ─────┼──→ CenterPoint 推理
                         │
warning.yaml ───────────┼──→ 预警逻辑
                         │
system.yaml ────────────┴──→ ROS1 接口 / 日志 / 诊断
```

### 1.2 配置来源与更新频率

| 配置文件 | 来源 | 更新频率 | 热更新 |
|---|---|---|---|
| `calib.yaml` | 标定团队 | 标定完成后不动，长期可能微调 | 需重启 |
| `geometry.yaml` | 现场测量 + 图像标注 | 一次配置，长期稳定 | 需重启 |
| `warning.yaml` | 算法工程师配置 + 现场调试 | 现场调试阶段频繁 | **支持** |
| `system.yaml` | 部署工程师配置 | 部署时一次 | 部分支持 |
| `model/*.py` | 算法工程师 | 训练/迭代阶段 | 需重启 |

## 2. calib.yaml — 标定参数

**来源**: 标定团队
**加载方**: 数据预处理模块、CenterPoint DataLoader
**使用场景**:
- 点云拼接 (雷达间外参 `LIDAR_RE_to_LIDAR_FR`)
- 3D→2D 投影 (相机内参 + 相机-雷达外参)
- 相机融合扩展投影 (相机内外参 + 点云/地面投影)
- 图像去畸变

### 关键字段

```yaml
sensors:
  CAM_FR:                      # 传感器 ID
    type: camera
    intrinsic: {fx, fy, cx, cy}
    distortion: {model, coeffs}
    resolution: [1080, 1920]  # height,width; 竖屏
  LIDAR_FR:
    type: lidar
    frequency: 10

extrinsics:
  CAM_FR_to_LIDAR_FR: {R, T}   # 相机→雷达
  LIDAR_RE_to_LIDAR_FR: {R, T} # 雷达→雷达 (拼接用)
  LIDAR_FR_to_BASE: {R, T}     # 雷达→车体/BEV坐标系
```

详见 [标定需求](./标定需求.md) 第 5 节完整模板。

## 3. geometry.yaml — RTG 几何与车道配置

**来源**: 现场测量 + 相机图像黄线标注
**加载方**: 预处理模块 (自车点过滤)、预警模块 (距离/运动方向/场地先验)、运动估计模块

### 3.1 自车 footprint (`ego_footprint`)

4 条支腿在 BEV 坐标系下的坐标。两个独立矩形：
- `truck_lane_side`: 集卡侧前后腿 (y ≈ 0)
- `forbidden_side`: 禁行侧前后腿 (y ≈ -23.5m)

**消费方式**:
- 点云预处理: 落在 footprint 矩形内的雷达点被标记为"自车"，不参与检测
- BEV 可视化: 绘制自车 footprint 供操作员参考
- 预警: 计算目标到最近支腿的距离

### 3.2 车道边界 (`lanes`)

两级设计——**标注输入** + **自动计算结果**:

```
图像黄线像素标注 (手动一次)
  │  camera_front.yellow_lines / camera_rear.yellow_lines
  │  pixel_points: [[u1,v1], [u2,v2], ...]
  ▼
image_yellow_lines_to_bev.py (自动脚本)
  │  读取 calib.yaml 内外参 → 投影到 z=0 平面
  ▼
bev_boundaries (自动填充)
  │  main_lane / truck_lane / forbidden_side_zone
  ▼
可选场地先验消费
```

### 3.3 自车遮挡 mask (`self_occlusion`)

相机画面中固定被本车支腿/设备罩遮挡的像素区域。用于:
- 数据质检时自动排除遮挡区域
- 标注辅助 (标注工具中灰色显示 mask 区域)

## 4. warning.yaml — 预警参数

**来源**: 算法工程师 + 现场调试
**加载方**: 预警模块 (warning_engine)
**热更新**: **支持** (修改后重载，不需要重启系统)

### 4.1 帧确认策略 (`frame_confirmation`)

```
危险等级: 1 帧即时触发
  目标进入 danger 距离 → 立刻输出 danger (不等待)
警告/提示: 连续 3 帧确认
  目标进入 warning/info 距离 → 连续 3 帧满足距离和可选场地先验 → 才触发
解除: 延迟 3 帧消退
  目标离开阈值范围 → 连续 3 帧都不满足触发条件 → 才解除预警
```

### 4.2 距离阈值 (`distance_thresholds`)

各类别三级距离阈值，距离定义为**目标框中心到最近 RTG 支腿的 BEV 平面投影距离**。

| 类别 | danger | warning | info |
|---|---|---|---|
| person | 8m | 15m | 25m |
| truck | 12m | 25m | 40m |
| car | 10m | 20m | 35m |
| other_obstacle | 10m | 20m | 35m |

### 4.3 ROI 区域权重 (`roi_zones`) [已废弃]

> **预警已改为距离 + 运动方向驱动，不再依赖固定车道/ROI 区域。**
> `roi_zones` 现已清空，`site_layout` 仅作为可选先验用于集卡车道降权。
> 详见 geometry.yaml §5 site_layout。

### 4.4 热更新接口

预警配置支持通过 ROS1 service 在线修改，无需重启:

```
# 修改人员危险距离阈值
rosservice call /rtg_bev/set_warning_param "key: 'distance_thresholds/person/danger', value: '6.0'"

# 重载整个 warning.yaml
rosservice call /rtg_bev/reload_warning_config "{}"
```

## 5. system.yaml — 系统/部署配置

**来源**: 部署工程师
**加载方**: 主程序入口 (ROS1 node)

### 5.1 ROS1 topic 配置 (`ros.subscribers`, `ros.publishers`)

所有输入/输出 topic 名称，不硬编码。现场 topic 名与设计不同时只需改此文件。

### 5.2 传感器配置 (`sensors`)

预期帧率、编码格式，仅用于诊断对比——不控制传感器。

### 5.3 模型配置 (`model`)

指向实际的 CenterPoint 配置文件和权重文件路径。

### 5.4 调试开关 (`debug`)

三个调试可视化 topic:
- `bev_visualization`: BEV 鸟瞰图 (检测框 + ROI + footprint)
- `image_projection`: 图像投影叠加 (3D 框投影 + 点云投影)
- `pointcloud_debug`: 拼接后的点云 (调试标定)

开发/部署切换只需改 `debug.enabled: true/false`。

### 5.5 回放模式 (`replay`)

`replay.enabled: true` 时从 rosbag 回放而非在线订阅，用于离线测试和复盘。

## 6. 模型配置 — rtg_centerpoint_voxelnet.py

**来源**: 算法工程师 (基于 CenterPoint 官方配置改造)
**加载方**: CenterPoint (det3d) 训练/推理框架

（原 SparseFusion 改造方案已归档删除，详见 [总体技术方案](./总体技术方案.md) 第4节）。

关键配置项:

```python
point_cloud_range = [-60.0, -25.0, -3.0, 60.0, 25.0, 8.0]
voxel_size = [0.075, 0.075, 0.2]
num_camera_views = 2
class_names = ['person', 'truck', 'car', 'other_obstacle']
use_sweeps = False
img_scale = (1080, 1920)  # 竖屏输入
```

## 7. 配置加载流程

```
系统启动
  │
  ├─ 1. 加载 system.yaml
  │     读取 ROS topic 名、GPU ID、日志配置
  │
  ├─ 2. 加载 calib.yaml
  │     验证外参矩阵非奇异、内参 fx/fy > 0
  │
  ├─ 3. 加载 geometry.yaml
  │     验证 footprint 坐标在合理范围
  │     如果有 pixel_points 标注 → 自动计算 bev_boundaries
  │
  ├─ 4. 加载 warning.yaml
  │     验证阈值单调性: danger < warning < info
  │
  ├─ 5. 加载模型配置 (.py)
  │     初始化 CenterPoint 模型 + 加载权重
  │
  ├─ 6. 初始化 ROS1 node
  │     按 system.yaml 订阅/发布 topic
  │
  └─ 7. 进入主循环
        在线推理或 rosbag 回放
```

## 8. 配置验证

启动时自动检查：

| 检查项 | 条件 | 不满足时 |
|---|---|---|
| 外参矩阵有效 | det(R) ≠ 0, R 正交 | 报错退出 |
| 内参合理 | fx > 0, fy > 0 | 报错退出 |
| footprint 在 BEV 范围内 | -60 < x < 60, -25 < y < 25 | 警告 |
| 距离阈值单调 | danger < warning < info | 报错退出 |
| topic 名非空 | 字符串长度 > 0 | 警告 |
| 模型文件存在 | os.path.exists() | 报错退出 |
| GPU 可用 | torch.cuda.is_available() | 降级 CPU (如允许) |

## 9. 配置版本管理

```
config/
├── calib.yaml              # 不提交 (标定团队提供, 每台车不同)
├── calib_from_bag.yaml     # 提交 (bag 估计默认值)
├── calib_synthetic.yaml    # 提交 (Pipeline 验证用合成值)
├── geometry.yaml           # 提交 (默认设计值)
├── warning.yaml            # 提交 (默认预警参数)
├── system.yaml             # 提交 (默认开发环境配置)
├── rtg_all_sensors.rviz    # 提交 (RViz 布局)
└── model/
    └── rtg_centerpoint_voxelnet.py  # 提交 (训练配置)
```

- `calib.yaml` (真实标定值) 不提交，由标定团队在现场提供
- 其余 yaml 文件提交通用默认值，部署时按需修改
- 仓库中的 geometry/system/warning yaml 为开发/测试默认值
