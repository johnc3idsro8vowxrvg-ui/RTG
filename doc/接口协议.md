# RTG BEV 接口协议说明

> 本文档定义 RTG 大车防撞 BEV 感知系统的 ROS1 输入/输出接口，包括 topic 定义、消息格式和数据语义。

---

## 1. 概述

### 1.1 通信中间件
- **ROS1** (现场已确认使用 ROS1)
- 消息序列化: `roscpp` / `rospy` 标准

### 1.2 接口架构

```
输入 (订阅) — 传感器命名: 1=左前(集卡侧), 2=左后(集卡侧), 3=右前(禁行侧), 4=右后(禁行侧):
  /Camera_Raw_Img_01           → sensor_msgs/Image    (已确认)
  /Camera_Raw_Img_02           → sensor_msgs/Image    (待确认)
  /Camera_Raw_Img_03           → sensor_msgs/Image    (第一版仅录制)
  /Camera_Raw_Img_04           → sensor_msgs/Image    (第一版仅录制)
  /ouster1/points         → sensor_msgs/PointCloud2 (集卡侧前, 已确认)
  /ouster2/points         → sensor_msgs/PointCloud2 (集卡侧后, 已确认)
  /lidar3/rslidar3_points      → sensor_msgs/PointCloud2 (第一版仅录制)
  /lidar4/rslidar4_points      → sensor_msgs/PointCloud2 (第一版仅录制)
  (标定文件: YAML, 离线加载)

输出 (发布):
  /rtg_bev/detections          → 自定义 DetectionArray
  /rtg_bev/tracks              → 自定义 TrackArray
  /rtg_bev/warnings            → 自定义 WarningArray
  /rtg_bev/ego_motion          → 自定义 EgoMotionState
  /rtg_bev/diagnostics         → diagnostic_msgs/DiagnosticArray
```

## 2. 输入接口

### 2.1 相机图像

| 属性 | 值 |
|---|---|
| Topic | `/Camera_Raw_Img_01` (已确认), `/Camera_Raw_Img_02/03/04` (待确认) |
| 类型 | `sensor_msgs/Image` |
| 编码 | `bgr8` 或 `rgb8` (待确认) |
| 分辨率 | 海康相机，1920×1080 |
| 帧率 | ~10 Hz |
| 硬件同步 | 四相机硬同步 |

### 2.2 激光雷达点云

| 属性 | 值 |
|---|---|
| Topic | `/ouster1/points`, `/ouster2/points` |
| 类型 | `sensor_msgs/PointCloud2` |
| 字段 | `x, y, z, intensity, timestamp` (5 字段) |
| 帧率 | 10 Hz (OS1 128线和 RoboSense 32线实测均为 10 Hz) |
| 坐标系 | 各雷达自身坐标系 |

### 2.3 标定文件

不在线订阅，启动时从 YAML 文件加载。完整结构详见 [标定需求](./标定需求.md) 第 5 节模板。关键字段概要:

```yaml
# calib.yaml (结构统一见标定需求文档)
sensors:
  CAM_FR: {type: camera, intrinsic: {...}, distortion: {...}, resolution: [1920, 1080]}
  CAM_RE: {type: camera, intrinsic: {...}, distortion: {...}, resolution: [1920, 1080]}
  LIDAR_FR: {type: lidar, frequency: 20}
  LIDAR_RE: {type: lidar, frequency: 20}

extrinsics:
  CAM_FR_to_LIDAR_FR: {R, T}   # 相机→雷达 (4×4)
  CAM_RE_to_LIDAR_RE: {R, T}
  LIDAR_RE_to_LIDAR_FR: {R, T} # 雷达→雷达 (点云拼接用)
  LIDAR_FR_to_BASE: {R, T}     # 雷达→车体/BEV坐标系
  LIDAR_RE_to_BASE: {R, T}

ego_footprint:
  truck_lane_side:             # 集卡侧 footprint (传感器1/2侧, y≈0)
    front_leg: [0.0, 0.0]     # 前腿地面投影中心
    rear_leg: [-12.0, 0.0]     # 后腿地面投影中心
    leg_width: 1.0            # 腿宽 (m)
  forbidden_side:              # 禁行侧 footprint (传感器3/4侧, y≈-23.5m)
    front_leg: [0.0, 23.5]
    rear_leg: [-12.0, 23.5]
    leg_width: 1.0
```

## 3. 输出接口

### 3.1 检测结果 `/rtg_bev/detections`

**频率**: ≥10 Hz
**类型**: `rtg_bev_msgs/DetectionArray`

```yaml
Header header
Detection[] detections
```

**Detection 字段**:

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int32 | 检测目标序号 (逐帧递增，非跟踪 ID) |
| class_id | int32 | 0=person, 1=truck, 2=car, 3=other_obstacle |
| class_name | string | 类别名称 |
| confidence | float32 | 置信度 [0, 1] |
| x, y, z | float32 | 3D 框中心点 (BEV 坐标系，米) |
| w, l, h | float32 | 3D 框尺寸 (宽, 长, 高，米) |
| yaw | float32 | 朝向角 (弧度) |
| vx, vy | float32 | 速度估计 (可选，由跟踪器填充) |
| distance | float32 | 目标到 RTG 最近支腿的距离 (米) |

---

### 3.2 跟踪结果 `/rtg_bev/tracks`

**频率**: ≥10 Hz
**类型**: `rtg_bev_msgs/TrackArray`

```yaml
Header header
Track[] tracks
```

**Track 字段**:

| 字段 | 类型 | 说明 |
|---|---|---|
| track_id | int32 | 稳定跟踪 ID（跨帧一致） |
| class_id | int32 | 类别 |
| age | int32 | 跟踪持续帧数 |
| x, y, z | float32 | 当前估计位置 |
| vx, vy | float32 | 估计速度 (m/s) |
| w, l, h | float32 | 3D 框尺寸 |
| yaw | float32 | 朝向角 |
| state | int32 | 0=候选, 1=确认, 2=丢失 |

> 第一版 track_id 对内不对外（后续升级后启用此 topic）。

---

### 3.3 预警结果 `/rtg_bev/warnings`

**频率**: ≥10 Hz
**类型**: `rtg_bev_msgs/WarningArray`

```yaml
Header header
Warning[] warnings
uint8 ego_motion_state    # 0=静止, 1=运动+x, 2=运动-x, 3=未知
WarningZone[] active_zones # 当前激活的预警 ROI 区域
```

**Warning 字段**:

| 字段 | 类型 | 说明 |
|---|---|---|
| track_id | int32 | 关联的跟踪 ID |
| warning_level | int8 | 0=无风险, 1=提示, 2=警告, 3=危险 |
| target_class | int8 | 目标类别 |
| distance | float32 | 距离 (m) |
| zone | string | 所在区域: "lane_core" / "lane_approach" / "side_intrusion" |
| trigger_reason | string | 触发原因: "distance" / "intrusion" / "heading_toward" |
| timestamp | time | 触发时间戳 |

---

### 3.4 运动状态 `/rtg_bev/ego_motion`

**频率**: ≥10 Hz
**类型**: `rtg_bev_msgs/EgoMotionState`

| 字段 | 类型 | 说明 |
|---|---|---|
| state | uint8 | 0=静止, 1=运动+x, 2=运动-x, 3=未知 |
| confidence | float32 | 状态置信度 [0, 1] |
| displacement | float32 | 最近 1 秒估计位移 (m) |
| velocity_estimate | float32 | 估计速度 (m/s, 可选) |

---

### 3.5 系统诊断 `/rtg_bev/diagnostics`

**频率**: 1 Hz
**类型**: `diagnostic_msgs/DiagnosticArray`

**诊断项**:

| 诊断 Key | 说明 | 异常条件 |
|---|---|---|
| `sensor/camera/front` | 前相机帧率和时间戳 | 帧率 < 预期 50% / 时间戳跳跃 |
| `sensor/camera/rear` | 后相机 | 同上 |
| `sensor/lidar/os1_front` | 前雷达 | 帧率 < 15 Hz / 时间戳异常 |
| `sensor/lidar/os1_rear` | 后雷达 | 同上 |
| `inference/latency` | 模型推理延迟 | >150 ms |
| `inference/fps` | 系统帧率 | <10 Hz |
| `system/calib` | 标定文件状态 | 文件缺失/加载失败 |
| `system/gpu` | GPU 状态 | 显存不足 / 温度过高 |

> 第一版仅记录诊断日志，不做实时告警。以上字段为预留。

## 4. 坐标系约定

### 4.1 BEV 坐标系
- 原点: 集卡侧 OS1 前雷达在地面的垂直投影
- x: 大车道方向 (+x 为前方)
- y: 跨距方向 (+y 从禁行侧指向集卡侧)
- z: 竖直向上
- 所有输出检测结果和预警均使用此坐标系

### 4.2 ROS tf 树 (建议发布，非强制)

```
rtg_base_link
├── lidar_os1_front
├── lidar_os1_rear
├── camera_front_optical
├── camera_rear_optical
└── bev_origin (BEV 坐标系原点)
```

### 4.3 frame_id
- 检测/跟踪/预警: `rtg_bev_origin`
- 运动状态: `rtg_bev_origin`
- 诊断: `rtg_base_link`

## 5. 部署模式数据流

### 5.1 在线推理数据流

```
ROS1 Subscriber Callbacks:
  camera_front_cb(image)
  camera_rear_cb(image)
  lidar_front_cb(points)
  lidar_rear_cb(points)
      │
      ▼
  Synchronizer (时间戳对齐, 容忍窗口 ~50ms)
      │
      ▼
  Preprocessor:
    - 点云外参拼接 (rear → front 坐标系)
    - 点云 self-footprint 过滤 (去除自车支腿上的点)
    - 图像 resize + normalize
    - SparseDepth 生成
      │
      ▼
  CenterPoint Inference (GPU)
      │
      ▼
  Postprocessor:
    - NMS + 置信度过滤
    - 短时跟踪 (内部)
    - 运动状态推断 (静态背景 ICP)
    - 预警逻辑
      │
      ▼
  ROS1 Publisher:
    detections_pub.publish(...)
    warnings_pub.publish(...)
    ego_motion_pub.publish(...)
    diagnostics_pub.publish(...)
```

### 5.2 离线回放模式

```
ROS1 rosbag play → 同上数据流 → 输出结果另行录制为 bag 或日志
```

## 6. 消息包定义

### 6.1 ROS1 自定义消息包

建议创建 `rtg_bev_msgs` 包:

```
rtg_bev_msgs/
├── msg/
│   ├── Detection.msg
│   ├── DetectionArray.msg
│   ├── Track.msg
│   ├── TrackArray.msg
│   ├── Warning.msg
│   ├── WarningArray.msg
│   ├── WarningZone.msg
│   └── EgoMotionState.msg
├── CMakeLists.txt
└── package.xml
```

### 6.2 消息定义示例

**Detection.msg**:
```
int32 id
int32 class_id
string class_name
float32 confidence
float32 x
float32 y
float32 z
float32 w
float32 l
float32 h
float32 yaw
float32 vx
float32 vy
float32 distance
```

**Warning.msg**:
```
int32 track_id
int8 warning_level
int8 target_class
float32 distance
string zone
string trigger_reason
time trigger_time
```
