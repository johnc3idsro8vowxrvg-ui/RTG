# RTG BEV 测试验收方案

> 本文档定义 RTG 大车防撞 BEV 感知系统的测试方案，包括离线评估和现场验收两大部分。

---

## 1. 测试策略

### 1.1 测试金字塔

```
         ┌──────────┐
         │  现场验收  │  ← 甲方最终验收，功能演示 + 测试用例通过率
        ┌┴──────────┴┐
        │  集成测试   │  ← 传感器→推理→预警 全链路联调
       ┌┴────────────┴┐
       │  模块测试     │  ← 各模块独立验证 (检测/跟踪/预警/运动估计)
      ┌┴──────────────┴┐
      │  离线基准评估   │  ← 500 帧验证集，mAP/NDS/召回率/误报统计
     └────────────────┘
```

### 1.2 验收判定优先级
1. **功能演示通过** (最高优先级)
2. 现场测试用例通过率
3. 离线评估指标 (作为技术参考，非硬性约束)

## 2. 离线评估

### 2.1 评估数据集

| 数据集 | 帧数 | 用途 |
|---|---|---|
| 验证集 | ~500 帧 | 超参调优、模型选择、定量评估 |
| 连续序列 | 5~10 片段 | 跟踪稳定性、预警延迟、运动估计准确性 |

### 2.2 评估指标

#### 核心指标

| 指标 | 目标 | 计算方式 |
|---|---|---|
| 关键目标召回率 (Recall) | ≥98% (50m内) | TP / (TP + FN)，IoU≥0.3 |
| FP/frame | 尽可能低 | 每帧平均误检数 |
| FP/min | 尽可能低 | 每分钟平均误检数 |
| 分距离段 FP | 尽可能低 | [0,10]m、(10,25]m、(25,50]m 分别统计 |
| 精确率 (Precision) | ≥80% | TP / (TP + FP) |

#### 辅助指标

| 指标 | 说明 |
|---|---|
| 自定义 mAP | 按距离分段 [0-10m, 10-25m, 25-50m]，IoU 阈值为 {0.3, 0.5} |
| 自定义 NDS | 不含速度项 (无 GT 速度)，仅含 mAP + mATE + mASE + mAOE + mAAE |
| 各类别 AP | person / truck / car / other_obstacle 分别统计 |
| 定位误差 (ATE) | 平移误差均值 (m)，≤0.5m@50m |
| 尺度误差 (ASE) | 尺寸误差均值，≤10% |
| 朝向误差 (AOE) | yaw 误差均值 (rad) |

### 2.3 距离分段评估

不同距离段分别统计召回率和精确率:

| 距离段 | 说明 | 召回率目标 |
|---|---|---|
| 0~10 m | 近距离，防撞最关键 | ≥99% |
| 10~25 m | 中距离，预警关注 | ≥98% |
| 25~50 m | 远距离，提前感知 | ≥95% |

### 2.4 类别专项评估

| 类别 | 评估重点 |
|---|---|
| person | 远距离小目标召回率、夜间/低照度召回率 |
| truck | 3D 框贴合度、载箱/未载箱均需检测 |
| car | 与 truck 的类别混淆率 |
| other_obstacle | 检测到即可，不严格考核定位精度 |

### 2.5 连续序列评估

| 指标 | 说明 | 目标 |
|---|---|---|
| 跟踪稳定性 | ID switch 次数 / 目标存在帧数 | <5% |
| 跟踪连续性 | 目标被持续跟踪的帧数占比 | >90% |
| 预警延迟 | 目标进入预警区域到首次触发预警的时间 | <500 ms |
| 预警抖动 | 预警等级在连续帧间频繁切换的次数 | 低 |

## 3. 现场测试用例

### 3.1 测试场景设计

#### TC-01: 集卡正常驶近（前方）

| 项目 | 内容 |
|---|---|
| 场景 | RTG 向前行驶，1 辆集卡从前方沿集卡车道驶近 RTG |
| 距离 | 集卡从 60m 外驶近至 5m |
| 预期 | 检测 truck 无漏检；距离 <40m 触发提示，<25m 触发警告，<12m 触发危险 |
| 通过标准 | 全程无漏检，预警等级随距离正确升级，无虚警 |
| 变体 | TC-01a: RTG 静止，集卡驶近 → 不预警 |
|      | TC-01b: 集卡载箱驶近 → 仍需正确检测 |
|      | TC-01c: 夜间/低照度 → 召回率不显著下降 |

#### TC-02: 集卡正常驶近（后方）

| 项目 | 内容 |
|---|---|
| 场景 | RTG 向后行驶，集卡从后方驶近 |
| 预期 | 同 TC-01，但预警方向为 -x |
| 通过标准 | 正确识别后方危险，前方不误报 |

#### TC-03: 人员进入集卡车道

| 项目 | 内容 |
|---|---|
| 场景 | RTG 行驶中，人员进入集卡车道或大车道核心区 |
| 距离 | 人员从 30m 外出现，走近至 3m |
| 预期 | 检测 person 无漏检；距离 <25m 触发提示，<15m 触发警告，<8m 触发危险 |
| 通过标准 | 最短距离 8m 内单帧即触发危险，无延迟 |

#### TC-04: RTG 静止

| 项目 | 内容 |
|---|---|
| 场景 | RTG 静止，集卡和人员进入检测范围 |
| 预期 | 正常检测目标，但预警等级全部为 0（无风险） |
| 通过标准 | 有检测输出但无预警输出 |

#### TC-05: 多目标同时存在

| 项目 | 内容 |
|---|---|
| 场景 | 2~3 辆集卡同时在场 + 人员走动 |
| 预期 | 所有目标独立检测和跟踪，预警分别计算 |
| 通过标准 | 目标不漏检，跟踪 ID 稳定 |

#### TC-06: 运动状态推断

| 项目 | 内容 |
|---|---|
| 场景 | RTG 静止→启动向前→匀速→减速→停止→向后行驶 |
| 预期 | ego_motion 状态正确切换 (静止/运动+x/运动-x) |
| 通过标准 | 状态切换延迟 <1s，无误判 |

#### TC-07: 静止目标误报抑制

| 项目 | 内容 |
|---|---|
| 场景 | 大车道旁停有静止车辆或集装箱（非移动目标） |
| 预期 | 检测到 but 不触发高等级预警（因为没有逼近趋势） |
| 通过标准 | 静止目标不触发危险/警告 |

#### TC-08: 系统性能

| 项目 | 内容 |
|---|---|
| 场景 | 正常运行，记录延迟和帧率 |
| 预期 | 预处理、点云拼接、模型推理、后处理、ROS 发布、端到端延迟均输出 P50/P95，输出帧率 ≥10Hz |
| 通过标准 | 监测 5 分钟，95% 帧满足已冻结的阶段延迟预算和帧率要求 |

### 3.2 测试环境要求

- 现场实际 RTG 运行环境。
- 传感器全部在线、标定已验收。
- 系统部署在工控机，连接可视化界面。
- 测试期间录制 rosbag，用于复盘和分析。

### 3.3 测试记录模板

```yaml

  - 提示触发距离: XX m

  - 危险触发距离: XX m
```

## 4. 模块级测试

### 4.1 检测模块

| 测试项 | 方法 | 通过标准 |
|---|---|---|
| 单帧推理正确性 | 输入 rosbag 单帧，检查输出 bbox 无 NaN/全零 | 100% 帧通过 |
| 分阶段延迟 | 100 帧 warm-up + 500 帧计时 | 输出预处理/拼接/推理/后处理/发布/端到端 P50/P95 |
| GPU 显存 | nvidia-smi 监控 | <工控机 GPU 显存上限 80% |
| 类别输出 | 检查各类别是否均能输出 | 4 类均有输出 |

### 4.2 跟踪模块

| 测试项 | 方法 | 通过标准 |
|---|---|---|
| 单目标跟踪 | 连续序列中 1 个集卡稳定跟踪 | ID switch=0 |
| 多目标跟踪 | 连续序列中 2~3 个目标 | 各目标 ID 不混淆 |
| 目标消失处理 | 目标驶出检测范围 | 正确标记为 lost / 删除 |

### 4.3 预警模块

| 测试项 | 方法 | 通过标准 |
|---|---|---|
| 分级触发 | 模拟目标在不同距离和运动方向下进入预警范围 | 等级正确切换 |
| 连续帧确认 | 普通警告需 2~3 帧确认 | 不因单帧假阳性触发 |
| 危险单帧 | 近距离目标 1 帧即触发 | 不延迟 |
| 离开消退 | 目标离开距离阈值或被场地先验降权 | 延迟 2~3 帧解除 |

### 4.4 运动估计模块

| 测试项 | 方法 | 通过标准 |
|---|---|---|
| 静止检测 | RTG 静止 rosbag 片段 | 连续输出"静止" |
| 运动检测 | RTG 运动 rosbag 片段 | 正确输出方向 |
| 方向切换 | 静止→运动→停止→反向运动 | 状态正确跟随 |

## 6. 缺陷等级

| 等级 | 定义 | 示例 | 验收阻塞 |
|---|---|---|---|
| P0 致命 | 影响安全的关键缺陷 | 5m 内集卡未检测 → 未预警 | 阻塞 |
| P1 严重 | 影响核心功能 | 预警等级错误、频繁漏检 | 阻塞 |
| P2 一般 | 影响体验 | 远距离偶尔漏检、轻微抖动 | 不阻塞但有修复计划 |
| P3 轻微 | 不影响功能 | 日志格式问题、非关键输出字段缺失 | 不阻塞 |
