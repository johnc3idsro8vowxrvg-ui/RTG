## 1. BEV 坐标系

```
原点: L1（集卡侧前支腿 OS1 128线雷达）在地面的垂直投影
+x:   大车道方向（C1 朝向方向）
+y:   跨距方向，禁行侧 → 集卡侧（垂直于大车道），与 ROS 惯例一致
+z:   竖直向上

RTG 可沿大车道双向行驶（+x 或 -x）。坐标系为固定车体系，不随行驶方向翻转。
使用 +x/-x 代替"前/后"（因为前/后随行驶方向变化），集卡侧(y≈0)/禁行侧(y≈-23.5)代替"左/右"。
```

| 参数 | 值 |
|---|---|
| 原点 | L1 地面投影 (0, 0, 0) |
| x 范围 | [-60, 60] m |
| y 范围 | [-25, 25] m |
| z 范围 | [-3, 8] m |
| 检测范围 | BEV y∈[-25,25] 内输出 3D/BEV 检测框；超出范围仅雷达检测，有障碍物则输出 BEV 框 |

### 2.1 集装箱沿 x 轴排列

每个箱列的集装箱沿 x 轴首尾相接排列，x 方向延伸数百米。RTG 沿 x 轴行驶以到达不同箱位。

**轮组间距 vs 集装箱长度**：

```
  x 轴 → (大车道方向, 集装箱首尾相接排列)
  
  x=+12m   x=+6m    x=0      x=-6m    x=-12m   x=-18m   x=-24m
    │        │     (前腿)   (后腿)      │        │        │
    ▼        ▼       ▼        ▼        ▼        ▼        ▼
  ┌────────┬────────┬────────┬────────┬────────┬────────┐
  │  40ft  │   20ft │ 20ft   │    40ft集装箱(12.19m)    │    ← 单列示意
  │ 12.19m │  6.06m │ 6.06m  │                          │
  └────────┴────────┴────────┴────────┴────────┴────────┘
                         ├─ 12.0m ─┤
                         │ 轮组间距 │
                         │最多覆盖1个│
                         │ 20ft箱  │
                         └─────────┘

  实际每列连续排列(首尾相接):
  ... [20ft][40ft][40ft][20ft][40ft][20ft][20ft][40ft]... → x方向延伸数百米
```

**支腿间可见的集装箱**：

```
  x=0 (前腿)                   x=-12.0 (后腿)
    │                             │
    ▼                             ▼
  ┌───────────────────────────────┐
  │       20ft 集装箱 (6.06m)     │  ← 一个 20ft 箱(6.06m)可放入 12.0m 轮组间距内
  └───────────────────────────────┘
  
  或：
  
  │←───── 12.0m 轮组间距 ─────→│
  ├──────────────────────────────┤
  │         40ft 集装箱           │────→ 延伸到支腿外
  │    (12.19m, 约等于 12.0m 轮组间距)      │
  └──────────────────────────────┘
```

| 尺寸 | 值 | 说明 |
|---|---|---|
| 轮组间距 | 12.0m | 前后支腿 x 方向距离 |
| 20ft 箱长 | 6.06m | 1 个 20ft 箱可放入轮组间距内 |
| 40ft 箱长 | 12.19m | 远超过轮组间距，箱体延伸至支腿外侧 |

> **对检测的影响**：集装箱是静态背景不应被检测。集装箱沿 x 方向延伸很远，为 RTG 运动估计提供了大量稳定的静态参考点。BEV 检测范围 x∈[-60,60]m 覆盖约 10 个 20ft 箱或 5 个 40ft 箱。

## 6. 相机 1 画面语义不变性

**设计目的**: 无论 RTG 在哪个箱区，同编号相机拍到的画面语义结构不变。

**原理**: 两个相邻箱区的车道布局关于 y=-4.25 镜像对称，且两个 RTG 均朝 +x 方向（C1 均朝 +x）。因此同编号相机在各自 BEV 系中看到完全相同的场景排列。

**C1/C2 图像从左到右**:

```
← 图像左侧 (+y方向)                          图像右侧 (-y方向) →
┌────────────┬────────────┬────────────┬────────────┬────────────┐
│  相邻箱区   │  中间车道   │ 自车大车道  │ 自车集卡道  │ 自车集装箱道 │
│ (集装箱+集卡)│ (两箱区间)  │ (本RTG左轮) │ (∥x, ~4.5m)│  (列6→1)   │
└────────────┴────────────┴────────────┴────────────┴────────────┘
```

## 7. 传感器 FOV

### 7.1 相机 — 海康 DS-2XC64ZZY-SHZJ-4mm

| 参数 | 值 |
|---|---|
| 分辨率 | 1080×1920（竖屏安装） |
| 编码 | bgr8 |
| HFOV（原始） | **87.6°**（对应 1920 px 长边，竖屏安装后沿大车道 x 纵深方向+高度） |
| VFOV（原始） | **46°**（对应 1080 px 短边，竖屏安装后沿跨距 y 方向） |
| 对角视场角 | 103.5° |
| 安装高度 | ~4.0 m |
| 安装朝向 | 支腿外侧，斜向下，**沿大车道 ±x 方向** |
| 频率 | 10 Hz（实测） |

**朝向（2026-05-20 修正）**:

```
相机全部沿大车道方向安装:

C1 (集卡侧·前支腿, x=0):   朝 +x 前方 → 看大车道前方
C2 (集卡侧·后支腿, x=-12.0): 朝 -x 后方 → 看大车道后方
C3 (禁行侧·前支腿, x=0):   朝 +x 前方 → 看禁行侧大车道前方
C4 (禁行侧·后支腿, x=-12.0): 朝 -x 后方 → 看禁行侧大车道后方

竖屏 FOV 分配:
  87.6° (1920px 长边, 画面上下) → 沿大车道 x 方向纵深 + 高度 (近→远)
  46°   (1080px 短边, 画面左右) → 沿跨距 y 方向展开 (横向覆盖)

俯视图中相机 FOV 锥角 = 46° (±23°)，比 RoboSense 32 线的 70° 更窄
```

**C1 画面语义**（朝 +x 前方）:

```
← 图像左侧 (+y 方向 = 集装箱侧)    图像右侧 (-y 方向 = 相邻箱区侧) →
┌────────────┬────────────┬────────────┬────────────┬────────────┐
│  相邻箱区   │  中间车道   │ 自车大车道  │ 自车集卡道  │ 自车集装箱道 │
│ (集装箱+集卡)│ (两箱区间)  │ (本RTG左轮) │ (∥x, ~4.5m)│  (列6→1)   │
└────────────┴────────────┴────────────┴────────────┴────────────┘
```

> 注: 无论 RTG 在哪个箱区，C1 画面语义不变（车道镜像对称保证）。

**地面视场估算**（斜向下，俯角 α≈30°，安装高度 h=4.0m）:

```
横向宽度 @ 距离 D (沿 x, 由 VFOV 46° 决定): w = 2 × D × tan(46°/2) ≈ 0.85 × D
  例: @x=10m → w≈8.5m, @x=30m → w≈25.5m, @x=50m → w≈42.5m

纵深范围 (沿 x, 由 HFOV 87.6° 和俯角 α 决定):
  最近地面可见: h × tan(90° - α - 87.6°/2) ≈ 0.3m (距支腿底部)
  最远地面可见: h × tan(90° - α + 87.6°/2) ≈ 50m+
  例 α=30°: 画面下边缘≈0.3m, 画面上边缘≈50m+
```

| 水平 FOV（旋转后） | **~70°**（沿大车道方向，±35°） |
| 垂直 FOV（旋转后） | **360°** |
| 安装高度 | ~1.5 m |
| 安装方式 | **逆时针旋转 90°**（垂直 360° + 水平 ~70°） |
| 单帧点数 | ~16,000-25,000 pts |
| 字段 | x, y, z, intensity |
| 频率 | 10 Hz（实测） |

**旋转 90° 的含义**:
```
原始安装:  水平 360° ∴ 垂直 ~70° (-55° 至 +15°)
旋转90°后: 垂直 360° ∴ 水平 ~70° (沿大车道 ±35°)

俯视图 FOV 锥角 = 70° (±35°), 比相机 46° 更宽
设计目的: 获取大车道方向精细扫描，而非更宽水平覆盖
  L3 (前支腿 x=0): 70° HFOV 沿 +x 大车道前方
  L4 (后支腿 x=-12.0): 70° HFOV 沿 -x 大车道后方
```

**地面覆盖估算**:
```
水平@距离D: 2 × D × tan(35°) ≈ 1.4 × D
  @10m: ±7m,  覆盖14m宽
  @30m: ±21m, 覆盖42m宽
  @50m: ±35m, 覆盖70m宽

垂直: 360° 环绕扫描，含地面和空中目标
  z=0平面距离: D_ground ≈ 1.5 / tan(φ)（取决于扫描线俯角）
  @1.5m高, 最密地面扫描线在雷达正下方半径~2-5m处
```

## 8. 相邻箱区对称关系

### 8.1 车道布局：轴对称

相邻箱区的**车道布局**关于中间车道中心线呈**轴对称（镜像对称）**：

**对称轴**: y = +4.25（中间车道中心线）

镜像公式: `y' = 2×(4.25) - y = 8.5 - y`（仅作用于 y 轴）

### 8.2 传感器位置：中心对称

相邻 RTG 的传感器与支腿在车道内的相对位置，与本 RTG 传感器呈**中心对称**关系（无固定中心点——因为 RTG 沿 x 轴运动，无法用一个固定坐标描述相邻 RTG 的绝对位置）。

本 RTG 传感器与相邻 RTG 对应传感器在各自车道内的关系示意：

| 本 RTG 传感器 | 位置 (本 BEV 系) | 相邻 RTG 对应传感器 | 位置特征 |
|---|---|---|---|
| C1 (前·集卡) | (0, 0) | 相邻 C1 (前·集卡) | 后腿位 + 镜像 y |
| C2 (后·集卡) | (-12.0, 0) | 相邻 C2 (后·集卡) | 前腿位 + 镜像 y |
| C3 (前·禁行) | (0, -23.5) | 相邻 C3 (前·禁行) | 后腿位 + 镜像 y |
| C4 (后·禁行) | (-12.0, -23.5) | 相邻 C4 (后·禁行) | 前腿位 + 镜像 y |

> 相邻 RTG 的 C1/C2（或 C3/C4）前后位置与本 RTG **互换**。此关系来自中心对称，但不绑定固定中心坐标——RTG 沿 x 轴运动，任何固定中心点都会随 RTG 位置改变。

### 8.3 车道布局镜像验证

| 本箱区 y 范围 | 镜像 y 范围 | 对应相邻箱区 |
|---|---|---|
| [-1.5, +1.5] | [+7.0, +10.0] | 相邻集卡侧大车道 |
| [-6.0, -1.5] | [+10.0, +14.5] | 相邻集卡车道 |
| [-22.63, -6.0] | [+14.5, +31.13] | 相邻 6 列集装箱 |
| [-25.6, -22.63] | [+31.13, +34.1] | 相邻禁行侧大车道 |

### 8.4 核心推论

| 性质 | 说明 |
|---|---|
| 相机语义不变 | 同编号相机在各自 BEV 系中看到相同的场景排列——车道镜像对称的直接结果 |
| 跨距不变 | 集卡侧↔禁行侧距离保持 23.5m |
| 传感器前后互换 | 相邻 RTG 与本 RTG 的传感器位置呈中心对称，C1/C2 和 C3/C4 前后互换 |
